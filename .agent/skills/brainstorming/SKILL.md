# Brainstorming

## When to use this skill
- When starting a new feature or "digital ecosystem" component.
- When an idea is vague and needs "precision engineering" before implementation.
- Before any complex "bespoke" logic is added to the system.

## Workflow
- [ ] **Research Fusion**: Check current project state (files, docs) to ensure integration.
- [ ] **Socratic Inquiry**: Ask questions one at a time to refine the idea.
- [ ] **Architecture Options**: Propose 2-3 approaches with trade-offs.
- [ ] **Sectioned Validation**: Present design in sections (200-300 words) and validate each.
- [ ] **Finalize Design**: Write validated design to `docs/plans/YYYY-MM-DD-<topic>-design.md`.

## Instructions

### 1. The Inquiry Phase
- **One question at a time**: Focus on understanding: purpose, constraints, and success criteria.
- **Value-Driven Context**: Frame questions around business impact and technical integrity.
- **Smart Tech Vocabulary**: Use terms like "Scalable Architecture," "Seamless Integration," and "Bespoke Logic."

### 2. Exploring Approaches
- Propose 2-3 different approaches (e.g., "Minimal Complexity" vs. "Future-Proof Fusion").
- Present options conversationally with your recommendation and reasoning.
- Focus on "Scalability" and "Integrity."

### 3. Presenting the Design
- Break the design into short, readable sections.
- Cover: architecture, components, data flow, and "Technical Integrity" (error handling/security).
- Ask "Does this look right so far?" after each section.

## Resources
- Consult `brand-identity` for visual and technical constraints.
- Hand off to the `planning` skill once the design is approved.
